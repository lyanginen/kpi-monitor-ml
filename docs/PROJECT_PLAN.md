# План разработки проекта

## Тема

Разработка системы мониторинга и анализа KPI сотрудников организации с использованием методов машинного обучения.

## Цель проекта

Разработать программный прототип системы, позволяющей хранить данные о сотрудниках и KPI, анализировать показатели эффективности и применять методы машинного обучения для прогнозирования риска снижения KPI.

## Основные модули

1. Модуль авторизации и ролей.
2. Модуль учета сотрудников.
3. Модуль учета отделов и должностей.
4. Модуль хранения KPI-показателей.
5. Модуль аналитики KPI.
6. Модуль машинного обучения.
7. Модуль отчетов.
8. Модуль настроек.
9. Административная панель.
10. Личный кабинет пользователя.

## Предварительные роли пользователей

1. Администратор.
2. Руководитель.
3. Сотрудник.

## Предварительные сущности базы данных

1. Пользователи.
2. Роли.
3. Сотрудники.
4. Отделы.
5. Должности.
6. KPI-показатели.
7. Записи KPI.
8. ML-прогнозы.
9. Метрики модели.
10. Отчеты.
11. Настройки.
12. Журнал действий.

## ML-задача

Предварительная задача машинного обучения: прогнозирование риска снижения KPI сотрудника на основе исторических данных.

Возможные типы задач:

- классификация риска: низкий, средний, высокий;
- регрессия итогового KPI на следующий период.
