# Тест-план проекта KPI Monitor ML

## 1. Идентификатор тест-плана

TP-KPI-MONITOR-ML-001

## 2. Введение

Настоящий тест-план описывает подход к тестированию десктопного приложения KPI Monitor ML.

Приложение предназначено для мониторинга и анализа KPI сотрудников организации с использованием методов машинного обучения.

## 3. Объект тестирования

Объектом тестирования является десктопное приложение KPI Monitor ML, включающее:

- модуль авторизации;
- модуль ролей пользователей;
- модуль сотрудников;
- модуль KPI-записей;
- модуль аналитики;
- ML-модуль;
- модуль отчетов;
- модуль настроек;
- админ-панель;
- личный кабинет;
- журнал действий.

## 4. Функции, которые будут протестированы

Будут протестированы следующие функции:

1. Запуск приложения через `main.py`.
2. Авторизация пользователей.
3. Разграничение доступа по ролям.
4. Просмотр списка сотрудников.
5. Просмотр карточки сотрудника.
6. Просмотр KPI-записей.
7. Добавление KPI-записи.
8. Просмотр аналитики KPI.
9. Обучение ML-модели.
10. Выполнение ML-прогноза.
11. Формирование DOCX-отчета.
12. Формирование XLSX-отчета.
13. Работа окна настроек.
14. Создание резервной копии базы данных.
15. Просмотр справки.
16. Просмотр журнала действий.

## 5. Функции, которые не будут протестированы

В рамках текущего этапа не тестируются:

- многопользовательская сетевая работа;
- промышленная эксплуатация под высокой нагрузкой;
- интеграция с внешними корпоративными системами;
- подключение к удаленной PostgreSQL-базе;
- проверка работы на всех возможных версиях Python.

## 6. Тестовые подходы

Для проекта выбраны следующие методы тестирования:

### 6.1 Позитивное тестирование

Проверка корректной работы приложения при вводе правильных данных.

Пример: вход под пользователем `admin / admin123`.

### 6.2 Негативное тестирование

Проверка поведения приложения при некорректных данных.

Пример: попытка входа с неправильным паролем.

### 6.3 Функциональное тестирование

Проверка соответствия функций приложения заявленным требованиям.

Пример: проверка формирования DOCX и XLSX отчетов.

### 6.4 Тестирование интерфейса

Проверка корректности отображения окон, таблиц, кнопок и сообщений.

Пример: проверка окна аналитики KPI.

### 6.5 Инсталляционное тестирование

Проверка запуска приложения в чистом виртуальном окружении после установки зависимостей из `requirements.txt`.

### 6.6 Регрессионное тестирование

Повторная проверка основных функций после внесения изменений в код.

## 7. Критерии начала тестирования

Тестирование начинается, если выполнены условия:

- проект загружен из Git-репозитория;
- создано виртуальное окружение Python;
- установлены зависимости из `requirements.txt`;
- создана база данных;
- база заполнена тестовыми данными;
- приложение запускается командой `python main.py`.

## 8. Критерии окончания тестирования

Тестирование считается завершенным, если:

- выполнены основные тест-кейсы;
- критические ошибки не обнаружены;
- приложение запускается;
- авторизация работает;
- основные окна открываются;
- отчеты формируются;
- ML-модель обучается;
- прогноз выполняется;
- результаты тестирования зафиксированы.

## 9. Критерии прохождения тестирования

Функция считается успешно протестированной, если фактический результат совпадает с ожидаемым результатом тест-кейса.

## 10. Критерии приостановления и возобновления тестирования

Тестирование приостанавливается, если:

- приложение не запускается;
- база данных повреждена;
- невозможно установить зависимости;
- критический модуль не импортируется.

Тестирование возобновляется после устранения блокирующей ошибки.

## 11. Результаты тестирования

Результатами тестирования являются:

- заполненные тест-кейсы;
- список найденных дефектов;
- баг-репорты;
- вывод о готовности приложения к демонстрации.

## 12. Ресурсы системы

Минимальные ресурсы:

- Python 3.10 или выше;
- 4 ГБ оперативной памяти;
- 1 ГБ свободного места;
- Windows / macOS / Linux;
- установленный пакетный менеджер pip.

## 13. Риски

| Риск | Вероятность | Влияние | Мера снижения |
|---|---|---|---|
| Ошибка установки PySide6 | Средняя | Высокое | Использовать актуальную версию Python |
| Несовместимость путей Windows/macOS | Низкая | Среднее | Использовать pathlib |
| Ошибка запуска ML-модуля | Средняя | Среднее | Проверить зависимости scikit-learn |
| Отсутствие файла БД | Средняя | Высокое | Запустить init_db и seed_data |
| Ошибка формирования отчетов | Средняя | Среднее | Проверить python-docx и openpyxl |

## 14. Ответственность

Разработчик выполняет:

- разработку приложения;
- первичное тестирование;
- отладку;
- исправление найденных ошибок;
- подготовку отчетных материалов.
