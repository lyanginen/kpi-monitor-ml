# Тест-кейсы KPI Monitor ML

## TC-001. Запуск приложения

**Предусловия:**

- Установлены зависимости из `requirements.txt`.
- База данных создана.
- База заполнена тестовыми данными.

**Шаги:**

1. Открыть терминал в корне проекта.
2. Выполнить команду `python main.py`.

**Ожидаемый результат:**

Открывается окно авторизации приложения KPI Monitor ML.

---

## TC-002. Успешная авторизация администратора

**Предусловия:**

- Приложение запущено.
- Открыто окно авторизации.

**Шаги:**

1. Ввести логин `admin`.
2. Ввести пароль `admin123`.
3. Нажать кнопку «Войти».

**Ожидаемый результат:**

Открывается главное окно приложения. Пользователь имеет роль «Администратор».

---

## TC-003. Ошибка авторизации при неверном пароле

**Предусловия:**

- Приложение запущено.
- Открыто окно авторизации.

**Шаги:**

1. Ввести логин `admin`.
2. Ввести пароль `wrongpassword`.
3. Нажать кнопку «Войти».

**Ожидаемый результат:**

Появляется сообщение об ошибке авторизации. Главное окно не открывается.

---

## TC-004. Просмотр списка сотрудников администратором

**Предусловия:**

- Пользователь авторизован как администратор.

**Шаги:**

1. Открыть раздел «Сотрудники».
2. Просмотреть таблицу сотрудников.

**Ожидаемый результат:**

В таблице отображается список всех сотрудников организации.

---

## TC-005. Просмотр карточки сотрудника

**Предусловия:**

- Пользователь авторизован как администратор.
- Открыто окно списка сотрудников.

**Шаги:**

1. Выбрать сотрудника в таблице.
2. Нажать кнопку «Открыть карточку».

**Ожидаемый результат:**

Открывается карточка сотрудника с личными данными и историей KPI.

---

## TC-006. Просмотр KPI-записей

**Предусловия:**

- Пользователь авторизован.

**Шаги:**

1. Открыть раздел «KPI-записи».

**Ожидаемый результат:**

Открывается таблица KPI-записей, доступных текущему пользователю.

---

## TC-007. Добавление KPI-записи администратором

**Предусловия:**

- Пользователь авторизован как администратор.
- Открыто окно KPI-записей.

**Шаги:**

1. Нажать кнопку «Добавить KPI».
2. Выбрать сотрудника.
3. Выбрать KPI-показатель.
4. Указать период.
5. Указать фактическое значение.
6. Указать плановое значение.
7. Нажать «Сохранить».

**Ожидаемый результат:**

Новая KPI-запись сохраняется в базе данных и отображается в списке.

---

## TC-008. Запрет добавления KPI сотрудником

**Предусловия:**

- Пользователь авторизован как сотрудник.

**Шаги:**

1. Открыть раздел «Мои KPI».

**Ожидаемый результат:**

Кнопка добавления KPI недоступна обычному сотруднику.

---

## TC-009. Просмотр аналитики KPI

**Предусловия:**

- Пользователь авторизован.

**Шаги:**

1. Открыть раздел «Аналитика KPI» или «Аналитика отдела».
2. Перейти по вкладкам «Сводка», «По отделам», «По сотрудникам».

**Ожидаемый результат:**

Отображаются рассчитанные показатели KPI с учетом роли пользователя.

---

## TC-010. Обучение ML-модели

**Предусловия:**

- Пользователь авторизован как администратор.
- В базе есть KPI-записи.

**Шаги:**

1. Открыть раздел «ML-анализ».
2. Нажать кнопку «Обучить модель».

**Ожидаемый результат:**

Модель обучается, отображаются метрики Accuracy, Precision, Recall, F1-score. В папке `models` появляется файл `model.pkl`.

---

## TC-011. Выполнение ML-прогноза

**Предусловия:**

- ML-модель обучена.
- Файл `models/model.pkl` существует.

**Шаги:**

1. Открыть раздел «ML-анализ».
2. Нажать кнопку «Выполнить прогноз».

**Ожидаемый результат:**

В таблице отображаются прогнозы риска KPI для сотрудников.

---
