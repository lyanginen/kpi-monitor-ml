# Чек-лист приемки KPI Monitor ML

## Общие требования

- Проект находится в Git-репозитории.
- Количество коммитов не менее 50.
- В корне проекта есть `main.py`.
- В корне проекта есть `requirements.txt`.
- Приложение запускается командой `python main.py`.
- В проекте есть подробные комментарии на русском языке.

## База данных

- Используется СУБД с поддержкой SQL-запросов.
- База данных хранится в проекте.
- В базе данных не менее 10 таблиц.
- Есть таблицы пользователей, ролей, сотрудников, KPI, отчетов и ML-прогнозов.
- База заполнена тестовыми данными.

## Роли пользователей

- Есть роль администратора.
- Есть роль руководителя.
- Есть роль сотрудника.
- Доступ к разделам отличается в зависимости от роли.

## Интерфейс

- Реализовано не менее 10 оконных форм.
- Есть окно авторизации.
- Есть главное окно.
- Есть окно сотрудников.
- Есть карточка сотрудника.
- Есть окно KPI-записей.
- Есть форма добавления KPI.
- Есть окно аналитики KPI.
- Есть окно ML-анализа.
- Есть окно отчетов.
- Есть окно настроек.
- Есть окно справки.
- Есть админ-панель.
- Есть личный кабинет.

## Машинное обучение

- Реализовано обучение ML-модели.
- Используется библиотека scikit-learn.
- Модель сохраняется в файл.
- Файл модели находится в папке `models`.
- Отображаются метрики качества модели.
- Выполняется прогноз риска KPI.

## Отчеты и файлы

- Реализовано формирование DOCX-отчета.
- Реализовано формирование XLSX-отчета.
- Отчеты сохраняются в папку `exports`.
- Реализовано открытие папок приложения из интерфейса.
- Реализовано резервное копирование базы данных.

## Администрирование

- Есть админ-панель пользователей.
- Можно просмотреть список пользователей.
- Можно активировать и отключать учетные записи.
- Есть журнал действий пользователей.

## Тестирование

- Подготовлен тест-план.
- Подготовлены тест-кейсы.
- Подготовлен шаблон баг-репорта.
- Проведена проверка основных сценариев.
